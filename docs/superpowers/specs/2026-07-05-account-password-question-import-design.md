# 账号、密码可见性与教师题库导入设计

## 目标

在不改变现有三角色平台信息架构和蓝白视觉体系的前提下：

- 为教师、学生、管理员各新增 6 个可真实登录的演示账号；
- 为所有创建、重置和修改密码场景提供一致的显示/隐藏控制；
- 将教师题库批量导入升级为可下载模板、上传预检、定位错误并事务化入库的完整流程。

## 账号方案

新增账号使用连续且容易区分的用户名：`teacher01` 至 `teacher06`、`student01` 至 `student06`、`admin01` 至 `admin06`。现有 `teacher`、`student`、`admin` 账号保留。

账号由受控的服务端管理命令创建，密码使用符合平台强度规则的角色化独立初始密码。命令只在首次创建时输出凭据，不把明文密码写入仓库、数据库或日志。重复运行时跳过已存在账号，保证幂等。

教师和学生同时创建对应档案。创建一个“基础工程演示班”，将 6 名学生加入该班，并按编号建立 `teacher01` 对 `student01` 等一对一强绑定。6 个管理员只创建管理员用户记录。

所有新账号状态为正常，`must_change_password` 为真，首次登录后必须修改密码。

## 密码输入控件

新增复用的 `PasswordField` 组件，内部使用 Lucide `Eye` 和 `EyeOff` 图标切换 `password` 与 `text` 类型。按钮必须为 `type="button"`，提供“显示密码/隐藏密码”的可访问名称，并保持输入框尺寸不跳动。

组件覆盖以下场景：

- 首次登录修改密码的当前密码、新密码、确认密码；
- 管理员新增教师或学生的初始密码；
- 管理员“教师与学生绑定”向导中的教师初始密码；
- 管理员学生名单批量导入的统一初始密码；
- 管理员重置账号密码。

登录页已有同类功能，保持现状，不重复改造。

## 教师题库导入流程

题库管理页保留“新建题目”，并增加“批量导入”按钮。批量导入使用宽弹窗分为三个状态：

1. **准备文件**：下载官方 XLSX 模板；通过拖拽区或文件选择器上传 XLSX/CSV；展示支持格式、大小限制和字段要求。
2. **预检结果**：展示总行数、有效数和错误数；有效题目以紧凑表格预览；错误按行号显示字段和原因；存在错误时禁止确认入库，但允许返回重新上传。
3. **确认入库**：再次显示新增数量、章节分布和题型分布；确认后以单个数据库事务写入，成功后关闭弹窗并刷新题库。

模板字段为：章节、题型、题干、选项A、选项B、选项C、选项D、正确答案、解析、知识点、难度、分值。选择题要求有效选项和答案；非选择题允许选项为空。题型、难度、分值、必填字段、文件内重复题干、Excel 公式内容均在服务端校验。

## 接口与数据流

- `GET /api/teacher/question-import-template`：下载官方 XLSX 模板。
- `POST /api/teacher/questions/import-preview`：上传文件并返回标准化题目、统计和逐行错误。
- `POST /api/teacher/questions/import`：服务端重新校验预检数据并事务化写入。

导入题目使用 `source="teacher-import"`，`created_by` 记录当前教师。教师只能查看、编辑和删除本人导入或创建的题目；教材题库继续只读。

## 错误与安全

- 文件扩展名、MIME 类型、5MB 大小限制和公式注入在服务端检查；
- 任一提交项不合法时整批回滚，不产生部分入库；
- 重复题干返回明确行号，不静默覆盖；
- 账号创建命令不打印密码哈希、密钥或数据库连接信息；
- 密码显示状态只存在于当前浏览器组件状态，不持久化。

## 测试与验收

- 后端测试覆盖账号命令幂等性、三角色档案和绑定、题库模板、预检错误、成功导入与事务回滚；
- 前端测试覆盖密码眼睛切换、导入弹窗、预检统计、错误阻断和成功刷新；
- 运行 `npm run check` 完成前端、后端、部署测试和生产构建；
- 部署后验证 18 个新增账号状态、生产发布 SHA、健康接口，以及教师题库导入页面在桌面和移动端无溢出。
