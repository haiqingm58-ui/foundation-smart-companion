# 登录页图片阴影柔化设计

## 背景

登录页轮播图片底部目前使用接近黑色的深蓝渐变，登录卡片外围同时带有范围较大的投影。两层效果叠加后显得偏重，削弱了工程图片本身的清晰感。

## 目标

- 将图片底部的黑重感改为更柔和的学院蓝层次。
- 将登录卡片外围投影改得更短、更淡。
- 保留轮播标题与说明文字的可读性。
- 不改变图片、文案、布局尺寸、响应式断点或登录交互。

## 已确认方案

采用方案 A：柔和蓝色层次。

### 图片底部遮罩

将 `.carouselTint` 从两段式深色渐变改为三段式学院蓝渐变：

```css
linear-gradient(
  180deg,
  rgba(22, 73, 122, 0) 34%,
  rgba(22, 73, 122, 0.16) 62%,
  rgba(22, 63, 115, 0.72) 100%
)
```

渐变在图片下半部才逐步出现，底部使用带明确蓝色倾向的半透明色，而不是接近黑色的遮罩。白色标题、说明和轮播圆点保持原位。

### 卡片外围投影

将 `.authColumns` 的大范围投影改为：

```css
0 10px 24px rgba(53, 84, 115, 0.08)
```

投影只用于区分白色卡片与浅蓝背景，不再形成明显黑边或厚重悬浮感。边框、圆角和背景色保持不变。

## 响应式行为

桌面、超宽屏和移动端共用同一组视觉参数。此次修改不触碰现有的 `1500px × 900px` 大屏规则，也不改变移动端堆叠结构。

## 验证

- 自动化测试读取计算后的渐变和卡片投影，防止深色旧值回归。
- 运行现有登录布局测试，确认桌面、大屏、断点边界和移动端几何尺寸不变。
- 在 1440 × 1024 与 390 × 844 视口截图，并将修改前后截图放入同一张对比图检查。
- 检查验证码、角色切换、密码显示按钮、登录按钮和版权文字均正常。
- 完成全量前端、后端、部署脚本、构建和浏览器流程验证后再发布。

## 非目标

- 不更换轮播图片。
- 不调整 Logo、按钮或输入框阴影。
- 不移动轮播文字或登录表单。
- 不重做登录页配色和信息结构。
