# 土力学独立题库、知识点与组卷系统设计

## 1. 目标

在《基础工程》智慧学伴中新增独立的“土力学”课程题库，并将教师端升级为完整的知识点、题目、试卷和考试管理系统。学生端同时支持按章节或知识点随机练习，以及完成教师发布的指定试卷。

系统必须保留源题库中的题干、选项、答案、图片、表格和公式信息；随机练习只更新学习记录与知识点掌握度，不计入正式课程成绩；教师发布的试卷进入正式成绩体系。

## 2. 课程隔离

- 新增课程科目实体，首批包含“基础工程”和“土力学”。
- 土力学题库拥有独立章节、知识点、题目统计和练习入口，不混入《基础工程》现有 7 章。
- 所有知识点、题目、试卷、随机练习和成绩记录必须关联课程科目。
- 原有基础工程题目迁移到“基础工程”科目，已有作业和练习记录保持可用。

## 3. 知识点库

- 土力学初始提供约 50 个有限知识点，按原教学章节组织。
- 知识点包含名称、所属科目、章节、说明、状态、创建人和排序。
- 同一科目内知识点名称规范化后唯一，避免同义重复。
- 教师可以新增知识点；新增项进入同一科目公共目录。
- 知识点可改名、停用或合并。已有题目引用的知识点不可直接删除。
- 知识点列表显示关联题目数量，并可反向查看全部题目。

## 4. 题目与知识点关系

- 题目与知识点采用多对多关系。
- 每道可用题目必须选择 1 至 3 个知识点。
- 一个知识点可以关联任意数量题目。
- 服务端在新建、编辑、批量导入时统一校验 1 至 3 个知识点限制。
- 题库支持按科目、章节、知识点、题型、难度、来源和关键词筛选。

## 5. 题目模型

题目支持以下类型：

- 单项选择题：至少 2 个选项，唯一正确答案。
- 多项选择题：至少 2 个选项，一个或多个正确答案。
- 判断题：正确或错误。
- 填空题：一个或多个标准答案，可配置同义答案。
- 简答题：参考答案、评分点和可选字数上限，字数范围为 20 至 2000。
- 计算题：题干、公式或附图、数据表、参考结果、步骤评分点和单位要求。

通用字段包括科目、章节、题干、选项、正确答案、解析、难度、默认分值、附件、状态、来源、创建人和版本时间。

计算题默认由教师批改；AI 只能提供步骤识别和评分建议，不直接形成最终成绩。简答题可由评分点初评，低置信度结果进入教师复核。

## 6. 土力学 Word 题库导入

- 解析压缩包中的 4 个 DOCX 和说明文件。
- 按文档顺序读取段落、表格、内嵌图片和公式对象。
- 识别章节、题型、题号、选项、答案和连续答案段落。
- 内嵌图片以内容哈希命名，存入受控题库资源目录。
- 表格存为结构化行列数据，不直接保存未经清理的 HTML。
- 名词解释归入简答题，保留原题型说明。
- 判断题答案统一规范为布尔值；选择题答案统一规范为选项标签数组。
- 以规范化题干和选项生成去重指纹。重复题优先保留答案、解析和附件更完整的版本。
- 导入分为“解析预检”和“正式入库”两阶段，正式入库事务化且幂等。
- 生成导入报告，包含源题数、成功数、重复数、图片数、表格数、无答案数、无法识别数和人工复核清单。
- 共享导入题目为只读题库；教师可复制为自建题目后编辑。

## 7. 教师端信息架构

教师端新增或重构以下入口：

1. 知识点库
2. 题库管理
3. 组卷中心
4. 考试与作业
5. 批改与成绩分析

### 7.1 知识点库

- 科目和章节筛选。
- 知识点名称、说明、题目数和状态列表。
- 新增、改名、停用、合并和查看关联题目。

### 7.2 题库管理

- 列表展示题干摘要、题型、章节、1 至 3 个知识点、难度、分值和来源。
- 编辑器按题型动态显示选项、答案、字数限制、评分点、单位和附件控件。
- 知识点使用可搜索多选控件，选择上限固定为 3。
- 支持批量导入、复制共享题目、新建、编辑和删除本人题目。

### 7.3 组卷中心

- 试卷作为可重复使用的独立实体，与一次发布任务分开。
- 支持手动选题和按蓝图自动抽题。
- 自动组卷条件包含科目、章节、知识点、题型、难度、题量和目标分值。
- 组卷结果可调整题型分区、顺序、每题分值和总分。
- 实时显示知识点覆盖率、题型比例、难度分布和重复风险。
- 试卷可保存草稿、复制、生成新版本、预览、在线发布和导出。

### 7.4 考试与作业

- 试卷可发布给指定班级或学生。
- 可配置开始时间、截止时间、考试时长、是否允许重做、自动评分和是否立即显示答案。
- 发布时将试卷题目复制为不可变快照，后续修改题库不影响历史考试。

## 8. 学生端

### 8.1 随机练习

- 学生先选择科目，再选择按章节或按知识点抽题。
- 按知识点抽题时允许选择 1 至 3 个知识点。
- 可选题量为 5、10、20，也允许输入不超过可用题量的自定义数量。
- 可进一步筛选题型和难度。
- 同一练习会话内不重复题目，并优先避开近期已答对的题目。
- 抽题后保存题目快照，刷新或中途退出可以继续。
- 完成后生成得分、错题、解析和知识点掌握度变化。
- 随机练习不进入正式课程成绩，但保留练习记录。

### 8.2 我的试卷

- 显示待开始、进行中、已提交和已批改四种状态。
- 学生只能访问发布给本人或所属班级的试卷。
- 支持答案自动保存、中途继续、倒计时、截止时间和提交确认。
- 单选、多选、判断和填空由服务端自动评分。
- 简答和计算题提交后显示“等待教师批改”。
- 批改完成后按教师设置展示总分、逐题反馈、标准答案、解析和知识点得分。

## 9. 试卷导出

同一份试卷支持生成以下文件：

- Word 试题卷
- PDF 试题卷
- Word/PDF 答题卷
- Word/PDF 参考答案与评分标准

导出配置包括标题、科目、考试时间、总分、学生信息栏、题型分区、是否显示分值和是否显示答案。图片、公式和表格必须与题目保持在一起，分页时不得截断题干与附件。

## 10. 数据模型

新增或调整以下实体：

- `subjects`：课程科目。
- `knowledge_points`：有限知识点目录。
- `question_knowledge_points`：题目与知识点多对多关系。
- `questions`：增加科目、附件、字数上限、评分模式、状态和来源元数据。
- `papers`：可复用试卷定义。
- `paper_questions`：试卷题目、分区、顺序和分值。
- `assignments`：增加试卷来源和发布设置。
- `assignment_questions`：保留发布时题目快照关系。
- `practice_sessions` 与 `practice_session_questions`：随机练习会话和抽题快照。

迁移必须兼容现有数据，不删除已有题目、作业、提交或练习记录。

## 11. 评分与掌握度

- 客观题由服务端标准答案评分。
- 简答题按 Rubric 评分点初评并记录置信度，教师可覆盖结果。
- 计算题默认人工批改，支持步骤分和单位检查。
- 一道题关联多个知识点时，得分按题目配置的知识点权重分配；未配置时平均分配。
- 正式试卷成绩进入课程成绩；随机练习只更新知识点掌握度。

## 12. 权限与审计

- 共享题库和共享知识点对教师可见。
- 教师只能编辑或删除本人创建的题目和知识点；管理员可进行合并与全局管理。
- 教师只能向本人管理的学生或班级发布试卷。
- 学生不能通过 API 获取未提交试卷的正确答案和评分标准。
- 题目导入、知识点合并、试卷发布、导出和批改均写入操作日志。

## 13. 错误处理

- DOCX 无法解析时保留源文件名和位置，不静默丢题。
- 无答案、题型不明、图片缺失和重复冲突进入人工复核列表。
- 自动组卷题量不足时明确显示缺少的题型或知识点数量，不自动降低要求。
- 学生提交遇到网络错误时保留本地草稿并允许重试。
- 导出失败不影响已保存试卷和在线发布状态。

## 14. 测试与验收

- 数据迁移测试覆盖现有题目和作业兼容性。
- 后端测试覆盖知识点 CRUD、1 至 3 个关联限制、题型校验、随机抽题、组卷、发布、权限、评分和幂等导入。
- DOCX 解析测试覆盖四份源文件的章节、题型、答案、图片和表格。
- 前端测试覆盖知识点多选、动态题型表单、组卷流程、学生随机练习和教师试卷作答。
- 导出文件必须进行 Word/PDF 页面渲染检查，确认中文、公式、图片、表格和分页正常。
- 生产部署后核对题目总数、知识点总数、重复与复核数量、线上随机练习、试卷发布、学生提交和导出下载。

## 15. 实施阶段

1. 数据模型、迁移、知识点与题目 API。
2. 教师知识点库和题目编辑器。
3. 土力学 DOCX 解析、预检、去重和正式导入。
4. 试卷模型、手动组卷和自动组卷。
5. 在线发布、学生答题、自动保存和评分。
6. Word/PDF 导出与视觉验证。
7. 全量回归、生产部署、数据核验和 GitHub 同步。

## 16. 非目标

- 本阶段不让 AI 自动给计算题形成最终成绩。
- 本阶段不提供监考摄像头、人脸识别或防作弊浏览器锁定。
- 本阶段不将随机练习成绩计入正式课程成绩。
